# PRP for P1.M6.T3.S3

## Objective

Create internal/generate/invariants_test.go — a white-box `package generate` test file that proves PRD §18.1 (the snapshot-based atomic-commit safety invariant) holds across EVERY failure path (parse-fail/rescue, timeout, dup-exhaustion, CAS-fail) and encodes static source-level tripwires (never `git commit`, never `update-ref --force`, never 1-arg update-ref except root). It CONSUMES the integration suite's in-package harness (P1.M6.T3.S2) — reusing newTempRepo/writeStage/seedCommit/e2eDeps/headMoverRunner/NewStubManifest + the assertion helpers — and adds: (1) IDEMPOTENT INDEX across all failure paths; (2) ATOMIC HEAD (Stagehand's failed CAS is a no-op — the stagehand commit is left dangling, HEAD is never force-advanced); (3) SNAPSHOT IMMUTABILITY (a write-tree tree object is byte-stable after additional files are staged mid-run); plus AST-based static guards over the production git source.

## Context

## Goal

**Feature Goal**: An executable proof of PRD §18.1 / §20.2 — the safety invariant that distinguishes Stagehand from every incumbent: across every failure path the repo is left byte-for-byte unchanged (modulo harmless dangling objects), and the production source is provably incapable of the unsafe ref mutations (`git commit`, `update-ref --force`, unguarded 1-arg update-ref).

**Deliverable**: One new file `internal/generate/invariants_test.go`, white-box `package generate`, NO new production code, NO build tag (runs in normal CI like integration_test.go). It compiles against the existing package and reuses the in-package test harness defined in integration_test.go / stubprovider_test.go.

**Success Definition**: `go test ./internal/generate/ -run 'TestInvariant|TestStatic'` is green; the full `go test ./internal/generate/` is green (no regression to the reused harness); `go vet` + `gofmt -s` clean on the new file. Every property is asserted against the REAL git binary via the REAL *git.Git / *provider.Executor — the ONLY mock is the stub agent binary.

## Why

- This is the product's defensibility core (PRD §13; decisions.md §7). The §18.1 invariant is a *marketing-grade* claim ("your repo is never mutated unless the commit succeeds") and these tests are its auditable proof.
- The integration suite (S2) already asserts these properties *inline* per scenario; S3 factors them into a dedicated, exhaustive property file so a regression to the invariant fails loudly and specifically.
- Static tripwires prevent a future contributor from reintroducing `git commit` porcelain or `--force` (the exact bugs the plumbing was designed to structurally prevent).

## What

A `package generate` test file that:
1. **IDEMPOTENT INDEX** (table-driven across ALL failure paths): snapshot `git diff --cached --name-only` BEFORE CommitStaged runs, drive a failing path, assert byte-identical AFTER.
2. **ATOMIC HEAD** (CAS-fail path): the failed `update-ref` is a no-op — the stagehand commit object exists but is DANGLING (unreachable from HEAD), and HEAD is NOT force-advanced to the stagehand commit.
3. **SNAPSHOT IMMUTABILITY**: a `write-tree` tree object is byte-stable (`git cat-file -p <TREE_SHA>` identical) even after additional files are staged mid-run.
4. **STATIC TRIPWIRES** (AST over production git source): never `git commit`, never `--force`, 1-arg update-ref only reachable via the root-commit guard.

### Success Criteria
- [ ] Index byte-identical before==after on parse-fail, timeout, dup-exhaustion, AND CAS-fail paths.
- [ ] CAS-fail leaves the stagehand commit dangling and HEAD at the concurrent mover's commit (no `--force`).
- [ ] A frozen write-tree tree is byte-stable after concurrent staging (and a fresh write-tree yields a different SHA).
- [ ] AST static scan of non-test `internal/git/*.go` finds no `"commit"` and no `"--force"` string-literal git args, and the single `update-ref` site is guarded by `if expected != ""`.

### Implementation Patterns & Key Details
```go
beforeStaged := stagedFiles(t, dir)   
if got := stagedFiles(t, dir); got != beforeStaged {
    t.Errorf("%s: IDEMPOTENT-INDEX violated: before=%q after=%q", name, beforeStaged, got)
}
afterHead := headSHA(t, dir)
if subj := strings.TrimSpace(gitRun(t, dir, "log", "-1", "--format=%s", afterHead)); subj != "concurrent commit elsewhere" {
    t.Errorf("ATOMIC-HEAD: HEAD subject=%q; want the decorator's commit (Stagehand must NOT force-advance)", subj)
}
allSubjects := gitRun(t, dir, "log", "--format=%s", "HEAD")  
if strings.Contains(allSubjects, "feat: ok message") {
    t.Errorf("ATOMIC-HEAD: the stagehand commit is reachable from HEAD (CAS must not have advanced it)")
}
fset := token.NewFileSet()
f, err := parser.ParseFile(fset, name, src, parser.ParseComments)
ast.Inspect(f, func(n ast.Node) bool {
    if cl, ok := n.(*ast.CompositeLit); ok {
        for _, el := range cl.Elts {
            if lit, ok := el.(*ast.BasicLit); ok && lit.Kind == token.STRING {
                args = append(args, strings.Trim(lit.Value, "\""))
            }
        }
    }
    return true
})

```
