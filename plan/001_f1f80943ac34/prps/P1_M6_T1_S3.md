# PRP — P1.M6.T1.S3: internal/generate/rescue.go — rescue protocol (FR43–FR45)

## Goal

**Feature Goal**: Implement the user-facing rescue protocol — the exact message Stagehand prints when, after safely snapshotting the staged index into the git object store, generation ultimately fails to produce a committed result (FR43 condition: TREE_SHA set and NEW_SHA not). The message tells the user (a) generation failed, (b) their files were safely snapshotted, (c) the frozen Tree ID, and (d) the EXACT copy-pasteable `git commit-tree ... | xargs git update-ref HEAD` command to recover manually — and, when a candidate message was rejected, hands them that message so their wait was not wasted (PRD §18.3 enrichment). This is the safety/transparency half of the product's defensibility claim (PRD §13, G10): a failed generation NEVER corrupts the repo and ALWAYS leaves the user a clear manual path. Rescue is a PURE RENDER function over the injected `*ui.Output` (no git binary, no exec, no os.Exit, no return value) — the loop control / signal wiring / exit-code setting are the CALLER's job (CommitStaged P1.M6.T1.S1, signal handler P1.M6.T2.S1).

## Why

- **This is the safety guarantee made legible (PRD §13 invariant + G10).** The repo-integrity invariant (§18.1) says refs/index are touched ONLY at the final `update-ref`, so any non-CAS failure leaves the repo byte-for-byte unchanged. But the user has a frozen TREE object and possibly a dangling commit in the object store — Rescue is what makes that SAFETY OBSERVABLE and ACTIONABLE: it hands them the Tree ID and the exact plumbing command to finish the commit by hand. Without it, a failure is a silent dead end.
- **FR51 stream discipline is structural here.** The rescue block is a FAILURE/RECOVERY notice, NOT a commit result. `ui.Output` (P1.M1.T2.S2) routes `Resultf`→stdout (ONLY the `[sha] subject`+diff-tree commit result) and `Progressf`→stderr (all human-facing diagnostics). Rescue MUST use `Progressf` (stderr). Routing it to stdout would corrupt `stagehand --dry-run | tee /tmp/msg.txt` pipelines (FR51). A rescue test MUST assert the block lands on stderr and stdout stays clean.
- **The candidate-message enrichment is a real UX win (PRD §18.3).** On duplicate-exhaustion or parse-failure-with-a-message, the agent already produced SOMETHING the user spent quota on. PRD §18.3: hand them `A candidate message was produced but rejected: "<msg>". You can use it manually in the command above.` so the wait was not wasted. The `candidate` parameter is exactly that rejected message ("" when none).
- **Root-commit correctness is an explicit enrichment over the proven script.** reference_impl.md §5 always printed `git commit-tree -p $PARENT_SHA ...`. But for an unborn repo (git.RevParseHEAD returns hasParent=false, PARENT_SHA=""), showing `-p <empty>` is wrong. The work-item contract requires: parent=="" → command omits `-p` entirely. This is a deliberate improvement the Go port makes over the zsh baseline.
- **Pure render = trivially and exhaustively testable.** No git binary, no exec, no filesystem, no time — inject a `*ui.Output` wired to a `bytes.Buffer`, call Rescue, assert substrings. This mirrors the codebase's pure-function table-test precedents (internal/provider/parse_test.go, internal/prompt/payload_test.go, and now internal/generate/dedupe_test.go).

```go
package generate
import (
	"fmt"
	"github.com/dustin/stagehand/internal/ui"
)
func Rescue(out *ui.Output, tree, parent, candidate string) {
	cmd := fmt.Sprintf("git commit-tree -m \"Your message\" %s | xargs git update-ref HEAD", tree)
	if parent != "" {
		cmd = fmt.Sprintf("git commit-tree -p %s -m \"Your message\" %s | xargs git update-ref HEAD", parent, tree)
	}
	out.Progressf("%s\n", out.Red("❌ Commit generation failed."))
	out.Progressf("------------------------------------------------------------\n")
	out.Progressf("Your staged files were safely snapshotted before generation.\n")
	out.Progressf("Tree ID: %s\n\n", tree)
	out.Progressf("To commit the originally staged files manually:\n")
	out.Progressf("  %s\n\n", cmd)
	out.Progressf("(omit \"-p <parent>\" if this is the repository's first commit)\n")
	out.Progressf("------------------------------------------------------------\n")
	if candidate != "" {
		out.Progressf("A candidate message was produced but rejected: \"%s\". You can use it manually in the command above.\n", candidate)
	}
}
```

```go
func Rescue(out *ui.Output, tree, parent, candidate string) {
	cmd := fmt.Sprintf("git commit-tree -m \"Your message\" %s | xargs git update-ref HEAD", tree)
	if parent != "" {
		cmd = fmt.Sprintf("git commit-tree -p %s -m \"Your message\" %s | xargs git update-ref HEAD", parent, tree)
	}
	out.Progressf("%s\n", out.Red("❌ Commit generation failed.")) 
	out.Progressf("------------------------------------------------------------\n")
	out.Progressf("Your staged files were safely snapshotted before generation.\n")
	out.Progressf("Tree ID: %s\n\n", tree)
	out.Progressf("To commit the originally staged files manually:\n")
	out.Progressf("  %s\n\n", cmd)
	out.Progressf("(omit \"-p <parent>\" if this is the repository's first commit)\n") 
	out.Progressf("------------------------------------------------------------\n")
	if candidate != "" { 
		out.Progressf("A candidate message was produced but rejected: \"%s\". You can use it manually in the command above.\n", candidate)
	}
}
```
