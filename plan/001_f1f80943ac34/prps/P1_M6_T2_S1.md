# PRP for P1.M6.T2.S1

## Objective

Create internal/generate/signal.go implementing the PRD §18.4 / reference_impl.md §1 signal contract for the snapshot-based atomic-commit pipeline: installSignalHandler(ctxCancel, rescueFn) arms a SIGINT/SIGTERM handler (post-snapshot) that cancels the executor context (process-group kill via Setpgid + grace SIGKILL), runs the rescue render if a snapshot was taken, and exits with ui.ExitRescue (3); restoreSignalHandler() resets SIGINT/SIGTERM to default and is called by CommitStaged immediately BEFORE UpdateRefCAS so a last-instant Ctrl-C is not mistaken for a failure. Deliver the file fully self-contained and unit/integration-tested, plus the two CommitStaged wiring call-sites applied only if generate.go already exists. This is a stdlib-only Go task (no testify), package generate, following the conventions of rescue.go/dedupe.go.

## Context

## Goal

**Feature Goal**: A tested, self-contained `internal/generate/signal.go` that implements the SIGINT/SIGTERM → cancel-and-rescue contract (PRD §18.4) for the atomic-commit pipeline, with the rescue-armed window open only between WriteTree and UpdateRefCAS.

**Deliverable**: `internal/generate/signal.go` exposing `installSignalHandler(ctxCancel, rescueFn func())` and `restoreSignalHandler()`, a testable pure core `handleSignal(...) int`, and an injectable `exit` seam; plus `internal/generate/signal_test.go` (pure unit tests + subprocess re-exec integration tests). The two wiring call-sites in `generate.go`'s CommitStaged are applied only if CommitStaged already exists at implementation time; otherwise the exact snippet is documented for P1.M6.T1.S1.

**Success Definition**: `go build`, `go vet`, `gofmt`, and `go test ./internal/generate/` all pass; a real SIGINT to a handler-armed process prints the §18.3 rescue block (with the frozen TREE_SHA) and exits 3; after restoreSignalHandler(), a SIGINT during a mocked update-ref does NOT print rescue and the path completes (exit != 3).

## Why

- **Safety invariant (PRD §18.1)**: the repo's refs/index change only at the final UpdateRefCAS. A Ctrl-C after the snapshot must NOT corrupt that — it must kill the agent cleanly and tell the user how to recover (rescue), without ever force-touching HEAD.
- **Defensibility**: process-group kill + grace SIGKILL + restore-before-commit is explicitly NEW PRD work (reference_impl.md §7 — "reference just trap'd"; §18.4) that hardens the proven shell pipeline.
- **Quota-not-wasted**: the rescue render (P1.M6.T1.S3) runs on the signal path so the user's spent agent quota is recoverable via the printed manual commit-tree command.

## What

### User-visible behavior
- After the index is snapshotted (write-tree) and before generation, a SIGINT/SIGTERM: (1) cancels the executor context → Executor.Run SIGTERM-then-SIGKILLs the agent's whole process group; (2) prints the §18.3 rescue block (tree SHA + manual recovery command) to stderr; (3) exits 3 (ui.ExitRescue).
- A SIGINT/SIGTERM BEFORE the snapshot uses Go's default disposition (exit 130) — no handler is armed yet (this realizes PRD §18.4 step 2's "else just exit").
- restoreSignalHandler() is called immediately BEFORE UpdateRefCAS: a Ctrl-C during the final atomic ref update is NOT mistaken for a generation failure (no rescue block; default disposition).

### Success Criteria
- [ ] installSignalHandler arms SIGINT+SIGTERM; on receipt it calls ctxCancel, then rescueFn, then exits ui.ExitRescue (3).
- [ ] restoreSignalHandler stops notification and resets SIGINT/SIGTERM to default (signal.Stop + signal.Reset).
- [ ] Ordering: restoreSignalHandler is invoked immediately BEFORE git.UpdateRefCAS (NOT after — see Gotcha #1).
- [ ] Pure core handleSignal is unit-testable with NO os.Exit (returns the exit code).
- [ ] Subprocess integration test: real SIGINT → rescue block printed with tree + exit code 3.
- [ ] Subprocess integration test: after restore, SIGINT during mocked update-ref → NO rescue block, exit code != 3.

## Implementation Blueprint

### signal.go structure (package generate, plain `package generate` line — generate.go owns the // Package doc comment, mirroring rescue.go/dedupe.go)
```go
package generate
import (
	"os"
	"os/signal"
	"syscall"
	"github.com/dustin/stagehand/internal/ui"
)
var exit = os.Exit
var commitSignals = []os.Signal{syscall.SIGINT, syscall.SIGTERM}
var (
	activeCh  chan os.Signal
)
func installSignalHandler(ctxCancel, rescueFn func()) {
	ch := make(chan os.Signal, 1)
	signal.Notify(ch, commitSignals...)
	activeCh = ch
	go func() {
		<-ch
		exit(handleSignal(ctxCancel, rescueFn))
	}()
}
func handleSignal(ctxCancel, rescueFn func()) int {
	ctxCancel()
	if rescueFn != nil {
		rescueFn()
	}
	return ui.ExitRescue
}
func restoreSignalHandler() {
	if activeCh != nil {
		signal.Stop(activeCh)
		activeCh = nil
	}
	signal.Reset(commitSignals...)
}

```
