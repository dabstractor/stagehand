# PRP — P1.M5.T1.S1: internal/config/config.go Config + internal/config/defaults.go (Default())

## Goal

**Feature Goal**: Create the NEW `internal/config` package — the resolved, in-memory `Config` type that carries ALL resolved settings plus the per-provider `ProviderOverrides` map, and `Default()` returning the built-in default values mandated by PRD §16.1 / decisions.md §6. This is the foundation of milestone M5: every later config task (the TOML/git-config/env/flag loaders in T2.S1, the precedence-chain `Load()` in T3.S1) and the downstream consumers (generate P1.M6.T1.S1, CLI P1.M7.T2) build on this exact type and its default table. The type is deliberately a clean, decoupled in-memory struct with zero go-toml coupling so the loader plumbing (T2.S1) can assemble it from tagged DTOs without touching this file.

**Deliverable**: The NEW `internal/config` package, four files:
- `internal/config/config.go` — OWNS `// Package config` (this is the first/primary file of a brand-new package; mirror internal/git/git.go and internal/ui/exitcode.go package-doc ownership). Defines the exported `Config` struct with the EXACT field set/types from the contract. Imports stdlib `time` + `github.com/dustin/stagehand/internal/provider` ONLY. NO `toml:` struct tags (D2). godoc on EVERY field citing the relevant PRD section.
- `internal/config/defaults.go` — plain `package config` with a file-level comment (mirror internal/git/log.go deferring to git.go). Defines the exported default constants centralizing the §16.1 table + `func Default() Config`.
- `internal/config/defaults_test.go` — white-box `package config`, stdlib `testing` + `time` ONLY.
- `internal/config/config_test.go` — white-box `package config`, stdlib `testing` + `time` + `github.com/dustin/stagehand/internal/provider` ONLY.

**Success Definition**: `go build ./internal/config/`, `go vet ./internal/config/` clean; `gofmt -l internal/config/` empty; `go test ./internal/config/` PASS; `go test ./...` whole-module green (git/provider/prompt/ui unaffected). `Default()` returns the EXACT §16.1 table (Timeout==120*time.Second, AutoStageAll==true, MaxDiffBytes==300000, MaxMdLines==100, MaxDuplicateRetries==3, Output==provider.OutputRaw=="raw", StripCodeFence==true, SubjectTargetChars==50) with the zero-valued fields (Provider=="", Model=="", Verbose==false, NoColor==false, ConfigPath=="", ProviderOverrides==nil). A populated Config round-trips every field including a ≥2-entry ProviderOverrides map. NO go.mod/go.sum change; NO go-toml import in the config package; NO import cycle (provider does not import config).

## Why

- **Config is the single resolved-settings type threaded through the whole binary.** decisions.md §2 + §6: every package (generate M6, prompt M4 callers, CLI M7) reads resolved settings off a `Config`. `Load()` (P1.M5.T3.S1) produces a `Config`; `Default()` is the lowest-precedence floor of the FR34 resolution chain. Defining the type + the default table FIRST lets the loaders (T2.S1) and `Load()` (T3.S1) be developed against a frozen contract.
- **ProviderOverrides is the user-override surface for the six built-in agent manifests.** PRD §16.1 bullet 7 + decisions.md §6: provider manifests merge FIELD-BY-FIELD (a user override setting only `default_model` keeps the rest of the built-in manifest). The merge already ships in `provider.NewRegistry(builtins, overrides)` (P1.M2.T3.S2, DONE). Config carries the overrides map; `Load()` (T3.S1) will call `provider.NewRegistry(provider.Builtins(), cfg.ProviderOverrides)`. Default() MUST leave it nil so the built-ins are NOT double-layered.
- **No import cycle.** plan_overview.md key decision 1: the Manifest type lives in `provider` (M2); `config` (M5) imports it; `provider/registry` does NOT import config. So `config → provider` is a clean one-way edge.
- **Decoupled from go-toml (D2).** A FLAT `Config` cannot be unmarshaled directly from the §16.2 file because scalars live under two TOML tables (`[defaults]` and `[generation]`). Adding `toml:` tags to a flat Config would be misleading; the tagged `defaultsDTO`/`generationDTO` → Config assembly is owned by P1.M5.T2.S1. This keeps T1.S1 a clean type definition with ZERO go-toml coupling.
