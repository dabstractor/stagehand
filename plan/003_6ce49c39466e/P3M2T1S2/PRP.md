## Goal

**Feature Goal**: Implement the FR-M2b one-file short-circuit in the Decompose orchestrator (PRD
§9.14 FR-M2b / §13.6.1 "One-file short-circuit"). In auto-decompose mode, when EXACTLY one path
differs between `baseTree` and the frozen `T_start`, bypass the planner agent ENTIRELY and produce a
single commit (the one file's frozen content) using the MESSAGE role to generate the message — saving
the planner round-trip that pure churn would incur (one file cannot be sensibly partitioned). An
explicit `--commits N` (N≥2) overrides the short-circuit. The check is deterministic (frozen
changed-path count), never model judgment.

**Deliverable** (1 file MODIFIED + tests; 0 NEW production files; 0 git.go changes):
1. `internal/decompose/decompose.go` — (a) the FR-M2b guard inserted in `Decompose()` between the
   freeze (line ~164) and `callPlanner` (line ~172): `if deps.Config.Commits == 0 && !deps.Config.Single
   { changedPaths := DiffTreeNames(baseTree, tStart); if len(changedPaths)==1 { return runOneFileShortcut(...) } }`;
   (b) the new `runOneFileShortcut(ctx, deps, preRunHEAD, isUnborn, baseTree, tStart)` function
   (placed near `runSingleShortcut`) that mirrors `runSingleShortcut`'s publish mechanics but ALWAYS
   generates the message via `generateMessage` (no planner message), commits `tStart` directly via
   `publishCommit`, and syncs the index via `ReadTree(tStart)` for a clean post-state; (c) FR-M2b doc
   comments.
2. `internal/decompose/decompose_test.go` — 4 core tests + 1 optional edge case.

## User Persona

**Target User**: the end user running `stagecoach` (no flags ⇒ auto-decompose) on a working tree with a
single changed file. The short-circuit is NOT a user-facing flag; it is an internal optimization that
skips a wasteful planner round-trip in the trivial case (PRD §13.6.1 FR-M2b / §9.14 FR-M2b).

**Use Case**: the user edits ONE file and runs `stagecoach`. Without FR-M2b, stagecoach invokes the
planner (which reads the whole diff, reasons, and returns `single:true` + a message — one agent
round-trip) even though one file can never be partitioned. FR-M2b detects the single file
deterministically and skips straight to message generation + commit — the same one commit, no planner
call. The user sees the same result, faster, with one fewer paid agent invocation.

**Pain Points Addressed**: wasted planner invocations (latency + cost) on the common single-file case;
the planner's N=1 decision is redundant when the changed-path count already proves N=1.

## Why

- **Implements PRD §9.14 FR-M2b / §13.6.1 "One-file short-circuit"** verbatim: "In auto-decompose mode,
  if the working tree has exactly one changed file … the planner is bypassed entirely … the same
  outcome as the FR-M11 single shortcut but with no planner agent call at all. A single file cannot be
  sensibly partitioned into multiple commits, so invoking the planner is pure churn; this check is
  deterministic (changed-path count), not model judgment. An explicit --commits N (N≥2) overrides this."
- **Deterministic, not model judgment (the load-bearing design property).** The count is taken over the
  FROZEN `T_start` (`DiffTreeNames(baseTree, tStart)`), NOT a live `StatusPorcelain` — so the
  short-circuit matches EXACTLY what the run would commit (FR-M1b). StatusPorcelain includes untracked
  `??` and is index-relative; `DiffTreeNames(baseTree, tStart)` is the frozen, committed-tree-relative
  change set (findings §6). This is the contract's point-1 research note.
- **Mirrors the proven `runSingleShortcut` (FR-M11) publish mechanics — lowest-risk pattern.** The
  publish tail (`publishCommit` → `buildCommitResult` → return) is IDENTICAL to FR-M11; only the message
  source differs (always `generateMessage`, no planner message). No new architectural concept.
- **Adds the index-sync (`ReadTree(tStart)`) that makes the post-state correct.** Committing `tStart`
  directly after the freeze (which reset the index to `baseTree`) would otherwise leave a STALE index
  (`MM` in `git status` — empirically verified, findings §4/§5). `runOneFileShortcut` syncs the index to
  `HEAD.tree` so `git status` is clean. (This is a latent gap in `runSingleShortcut` too — documented
  for a follow-up, NOT fixed here per scope.)

```go
// === Task 1: runOneFileShortcut — the planner-BYPASSED single-commit path (FR-M2b) ===
// runOneFileShortcut is the planner-BYPASSED single-commit path for AUTO mode when EXACTLY one path
// differs between baseTree and the frozen T_start (FR-M2b). A single file cannot be sensibly
// partitioned, so the planner is NEVER called (contrast runSingleShortcut/FR-M11, which DOES call the
// planner and uses its message). Here there is no planner message, so the message is ALWAYS generated
// by the MESSAGE role from the baseTree→tStart (tree-to-tree) diff. The committed tree is T_start
// directly (freeze-safe: NOT a live AddAll, which could sweep a concurrent change). The index is then
// synced to T_start (ReadTree) so `git status` is clean post-commit — the freeze reset the index to
// baseTree, and committing tStart directly would otherwise leave index==baseTree ≠ HEAD.tree==tStart
// (a stale `MM` state). *generate.RescueError (message) + *generate.CASError (CAS) propagate DIRECTLY.
// SIGNAL-FREE in S1.
func runOneFileShortcut(ctx context.Context, deps Deps, preRunHEAD string, isUnborn bool, baseTree, tStart string) (DecomposeResult, error) {
	msg, err := generateMessage(ctx, deps, baseTree, tStart) // the MESSAGE role (planner bypassed)
	if err != nil {
		return DecomposeResult{}, err // *RescueError — propagate DIRECTLY
	}
	newSHA, err := publishCommit(ctx, deps, tStart, preRunHEAD, msg) // commit T_start directly (freeze-safe)
	if err != nil {
		return DecomposeResult{}, err // *CASError / ErrPublicationFailed-wrapped — propagate DIRECTLY
	}
	// CRITICAL: sync the index to the committed tree (the freeze reset index→baseTree). tStart == HEAD.tree now.
	if err := deps.Git.ReadTree(ctx, tStart); err != nil {
		return DecomposeResult{}, fmt.Errorf("%w: one-file index sync: %w", ErrDecomposeFailed, err)
	}
	cr, err := buildCommitResult(ctx, deps, newSHA, msg, isUnborn) // isRoot == isUnborn (root commit iff unborn)
	if err != nil {
		return DecomposeResult{}, err
	}
	return DecomposeResult{Commits: []CommitResult{cr}, Amended: 0}, nil
}
```
